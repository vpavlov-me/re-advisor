# Review Queue

## Назначение

В эту папку попадают **выверенные между продактами эпики**, готовые к рассмотрению командой разработки.

## Процесс Review

### 1. Поступление эпиков
- Эпики переносятся сюда из `project-management/draft/` после согласования между продактами
- Эпики в этой папке считаются **бизнес-готовыми** (business requirements finalized)
- Все обязательные поля заполнены, приоритеты согласованы

### 2. Рассмотрение командой разработки
Команда разработки:
- Изучает эпики в папке `review/`
- Прогоняет через AI аналитика **Лану** для технической оценки
- Получает ответы на технические вопросы
- Оценивает сложность и техническую реализуемость
- При необходимости запрашивает уточнения у продактов

### 3. Переход в Epics Queue
После завершения review:
- **Если вопросов нет** → эпик переносится в `project-management/epics/`
- **Если есть вопросы** → создается документ `TECHNICAL-QUESTIONS.md` внутри эпика, вопросы отправляются продактам
- **После получения ответов** → эпик переносится в `project-management/epics/`

## Структура эпика в Review

Каждый эпик в review должен содержать:

```
epic-XXX-feature-name/
├── epic-overview.md              # Основное описание эпика
├── STORY-RECOMMENDATIONS.md      # Рекомендованные user stories
├── STAKEHOLDER-QUESTIONS.md      # Вопросы от бизнеса (если были)
└── user-journey-*.md             # User journey (опционально)
```

## Статусы эпиков

### В Review папке
```yaml
status: "in_review"              # Эпик находится на рассмотрении
review_stage: "technical"        # Техническая оценка командой
```

### После Review
```yaml
status: "ready_for_planning"     # Готов к планированию спринта
# Эпик перемещается в project-management/epics/
```

## Workflow

```
draft/ → review/ → epics/ → sprints/
   ↓         ↓         ↓         ↓
Продакты  Dev Team  Planning  Development
```

## Критерии готовности для перехода в Epics

- [ ] Техническая оценка завершена
- [ ] Все технические вопросы решены
- [ ] Story points оценены
- [ ] Технические риски идентифицированы
- [ ] Зависимости от других эпиков выявлены
- [ ] Архитектурные решения согласованы

## AI Аналитик Лана

**Лана** помогает команде разработки:
- Анализировать техническую сложность
- Выявлять потенциальные проблемы
- Предлагать архитектурные решения
- Оценивать объем работ
- Идентифицировать технические зависимости

### Типичные вопросы для Ланы:
```
> Проанализируй эпик EPIC-XXX на техническую реализуемость
> Оцени сложность реализации user stories в EPIC-XXX
> Выяви технические зависимости для EPIC-XXX
> Предложи подход к реализации функционала из EPIC-XXX
```

## Типичные технические вопросы

Команда разработки может запрашивать уточнения по:
- **Производительность:** Ожидаемая нагрузка, количество пользователей
- **Интеграции:** Внешние сервисы, API, зависимости
- **Безопасность:** Требования к аутентификации, авторизации
- **UX детали:** Поведение в edge cases, валидация данных
- **Приоритеты:** Что критично в MVP, что можно отложить

## Ответственность

| Роль | Ответственность |
|------|----------------|
| **Продакты** | Размещают эпики в review/, отвечают на технические вопросы |
| **Dev Team** | Проводят технический review, задают вопросы, оценивают сложность |
| **Tech Lead** | Финальное одобрение перехода в epics/ |
| **AI Лана** | Техническая аналитика, выявление рисков |
