## üìã Working with Epics and Sprints

### Understanding Epics
An **Epic** is a large feature that takes several sprints to complete. It contains:
- **Business Objectives:** Why we're building this
- **User Stories:** Who will use it and how
- **Acceptance Criteria:** How we know it's done
- **Success Metrics:** How we measure success

### Sprint Planning
Every 2 weeks, we plan a new sprint:
1. **Review completed work** from last sprint
2. **Plan new work** for upcoming sprint
3. **Identify blockers** and dependencies
4. **Set sprint goals** and priorities

### Your Role in Planning
- **Define requirements** - What should the feature do?
- **Write user stories** - Who will use it and why?
- **Set acceptance criteria** - How do we know it's done?
- **Prioritize features** - What's most important?
